# Unit TDD

This layer owns durable technical truth for one logical unit.

Use it for internal architecture, contracts, naming, sequencing, verification,
and package-local constraints that should survive directory refactors.

## Units

- `web/`: `@partner-up-dev/design-web`
- `uniapp/`: `@partner-up-dev/design-uniapp`

Local package `AGENTS.md` files remain the closest edit rules and tactical
hazard memory. Unit TDD owns slower-moving structure.
